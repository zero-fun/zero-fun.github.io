Spring源码分析(1)-IOC容器初始化过程分析

发表于2019-07-31|更新于2021-03-11
|阅读量:

首先给出 IOC 容器初始化的时序图

这里以 ClassPathXmlApplicationContext 为例子，加载 xml 文件中的对象。

大致流程分为 3 个步骤，定位，加载，注册

定位：定位 xml 所在位置

加载：将 xml 中的元素转化成 Resource 对象，加载对应的节点元素

注册：将 Resource 对象转化成 Document 对象，然后注册到 BeanDifinittion，保存到容器 beanDefinitionMap 中

实际分析过程

  1. 首先我们在加载一个 xml 配置文件的时候，会使用以下方式，所以我们以 ClassPathXmlApplicationContext 这个容器作为示例

    可以看出，我们是通过构造方法来加载配置文件的，那我们通过构造方法查看具体执行流程

    最终执行的构造方法为上述方法，然后我们追踪refresh()方法

  2. refresh方法如下：

    refresh方法完成了spring容器初始化的操作。

  3. 我们先定位IOC容器部分，也就是执行obtainFreshBeanFactory方法过程

  4. 继续跟踪refreshBeanFactory()方法

    refreshBeanFactory方法中首先判断容器是否存在，存在则销毁，然后创建一个新的DefaultListableBeanFactory对象来保存容器。

  5. 继续跟踪loadBeanDefinitions方法，加载bean定义规则

    创建了一个XmlBeanDefinitionReader读取器

    beanDefinitionReader.setResourceLoader(this);指定了资源加载器为当前对象（父类AbstractApplicationContext继承了DefaultResourceLoader，所以子类拥有资源加载的功能）

  6. 跟踪loadBeanDefinitions方法

    可以看到通过上面定义的resourceLoader将资源文件转化成Resource对象

  7. 继续往下追踪loadBeanDefinitions，最后调用doLoadBeanDefinitions方法

    将资源文件转化成Document对象，然后开始在注册beanDefinition

  8. 追踪registerBeanDefinitions方法

  9. 追踪parseBeanDefinitions方法，继续往下，调用processBeanDefinition方法

    我们可以发现委托BeanDefinitionParserDelegate将Element元素转化成了BeanDifinitionHolder对象，然后通过BeanDefinitionReaderUtils.registerBeanDefinition将bean放入到容器DefaultListableBeanFactory对象的beanDefinitionMap中
