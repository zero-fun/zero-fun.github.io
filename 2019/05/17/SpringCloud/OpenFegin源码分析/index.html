Openfegin 源码分析

在使用 fegin 的过程中，最重要的一点就是在 application 中增加@EnableFeignClients，来启用 feign。

我们来看看这个注解的声明。

在这个注解中，主要是通过 import 动态装载了 FeignClientsRegistrar 类，这个就是 spring 的动态装载 registrar 的实现，我们来看看 FeignClientsRegistrar 的具体实现

重写 registerBeanDefinitions 方法，注入 bean。

registerFeignClients 这个方法中就是扫描所有配置的包路径，然后找到所有被@FeignClient 注解的类，添加到 BeanDefinition

在 registerFeignClient 这个方法中，定义了一个 FeignClientFactoryBean 的工厂 bean，通过这个工厂 bean 把被@FeignClient 注解的类进行了封装。

我们来看看 FeignClientFactoryBean 的实现

FeignClientFactoryBean 实现了 FactoryBean 接口，这个接口最重要的方法 getObject 被 FeignClientFactoryBean 进行了重写。

在这个 getTarget 方法中，可以看到明显的封装请求 url 地址等相关参数的代码。

可以看到最后通过 target.target 方法返回了实例，跟踪到默认实现 DefaultTargeter 里面查看一下

进入 build 的 newinstance 方法

可以看到明显的动态代理的代码，可以知道调用的时候是通过动态代理生成的代理对象来调用的
